01Apply vs Save distinction is invisible

main_window.py · _build_toolbar

Apply → volatile RAM (lost on power cycle). Save → flash (persistent). This is the most consequential distinction in the app and is only documented in a hover tooltip on Apply. Users who never hover it will lose all config.

Current
self._apply_btn = QPushButton("Apply")
self._apply_btn.setToolTip("Send dirty config sections to device (RAM; auto-save ~5s)")
# …
self._save_btn = QPushButton("Save")
Fix
# 1. Rename so the distinction is in the label itself
self._apply_btn = QPushButton("Apply to device")
self._apply_btn.setToolTip("Writes changes to device RAM only. Power cycle will revert unless you also Save.")
self._save_btn = QPushButton("Save to flash")
self._save_btn.setToolTip("Persists the current device config to non-volatile flash.")

# 2. Add a persistent "unsaved" indicator in the toolbar
self._unsaved_label = QLabel("● unsaved")
self._unsaved_label.setStyleSheet("color: #f0a040; font-size: 11px; padding: 0 4px;")
self._unsaved_label.setVisible(False)
tb.addWidget(self._unsaved_label)

# 3. Toggle the indicator in _update_apply_enabled()
self._unsaved_label.setVisible(connected and dirty)

06Errors are ephemeral — vanish after 8s, no visual distinction

Current

Fix — colour errors distinctly and reset style on restore
def _on_error(self, message: str) -> None:
    self._conn_label.setStyleSheet("color: #f08040; font-weight: 500;")
    self._show_status(f"⚠ {message}", 10_000)

def _restore_connection_status(self) -> None:
    self._conn_label.setStyleSheet("")  # clear error colour
    if self._connected_port:

    else:
        self._conn_label.setText("Disconnected")
